Concepts used:

  • CLOSURES

    ~ creating private states and functions (e.g.: Hanoi status 'height' accessible through 'get_height' method,

    ~ not polluting global variable namespace (the whole 'towerOfHanoiGame.js' module)

  • TYPE AUGUMENTATION

    ~ New Number,Array and Function methods

  • CASCADING

    ~ Hanoi method 'moveBlock' returns the Hanoi after the block movement enabling expressions as: Hanoi.moveBlock(0,2).moveBlock(0,1)

  • LOGIC OPERATORS

    ~ the behaviour of '&&' and '||' and truthiness/falsiness of objects

  • RECURSION

    ~ hanoi solver 'showSolutionPath'

  • MEMOIZATION

    ~ memoized Hanoi solver 'showSolutionPath_memoized'

  • LAMBDA/ARROW FUNCTIONS

    ~ in 'mutatePath' funtion

  • PERFORMANCE TESTING

    ~ 'performance' and 'meanPerformance' Function type augumentations (can be used to test performance of 'getSolutionPath' functions) Example:

    h.showSolutionPath_memoized.performance(h,([]));
